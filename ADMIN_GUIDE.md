# 🔐 Керівництво адміністратора бота

## Налаштування адміністратора

### 1. Отримайте свій Telegram ID

1. Відкрийте [@userinfobot](https://t.me/userinfobot) в Telegram
2. Натисніть **Start**
3. Бот надішле вам ваш **User ID** (наприклад: `123456789`)
4. Скопіюйте цей номер

### 2. Додайте ID у конфігурацію

Відредагуйте файл `.env`:

```env
ADMIN_USER_IDS=123456789
```

**Для декількох адміністраторів** (через кому):
```env
ADMIN_USER_IDS=123456789,987654321,555666777
```

### 3. Перезапустіть бота

```powershell
# Зупиніть бота (Ctrl+C)
# Запустіть знову
python bot_appscript.py
```

Ви побачите:
```
INFO - Адміністратори: [123456789]
```

## Адмін-команди

### `/admin_status` — Перевірити статус бота

**Використання:**
```
/admin_status
```

**Результат:**
- ✅ Статус бота: АКТИВНИЙ
- 🛑 Статус бота: ЗУПИНЕНИЙ

---

### `/admin_stop` — Зупинити прийом заявок

**Використання:**
```
/admin_stop
```

**Що відбувається:**
1. Бот припиняє приймати заявки
2. Користувачі бачать повідомлення:
   ```
   ❌ Прийом заявок закритий, до наступного тижня.
   ```
3. Всі існуючі заявки збережені та доступні в Google Sheets

**Коли використовувати:**
- Закінчився тиждень розіграшу
- Потрібна технічна перерва
- Хочете вручну зупинити реєстрацію

---

### `/admin_start` — Відновити прийом заявок

**Використання:**
```
/admin_start
```

**Що відбувається:**
1. Бот знову починає приймати заявки
2. Користувачі можуть реєструватися

**Коли використовувати:**
- Після `/admin_stop` для відновлення роботи
- Початок нового тижня розіграшу

---

## Сценарії використання

### Сценарій 1: Закінчення тижня

**П'ятниця 16:00:**
- Бот автоматично зупиняється (згідно з налаштуваннями)

**Або раніше вручну:**
```
/admin_stop
```

**Понеділок ранок:**
```
/admin_start
```

---

### Сценарій 2: Термінова зупинка

**Причина:** Потрібна перерва або виявлено проблему

```
/admin_stop
```

Бот негайно припиняє приймати заявки.

---

### Сценарій 3: Перевірка стану

**Будь-коли:**
```
/admin_status
```

Швидко дізнаєтесь, чи працює бот.

## Логіка роботи бота

### Автоматичні обмеження

Навіть якщо адмін **не** зупинив бота, він автоматично не приймає заявки:

1. **Вихідні (субота-неділя):**
   ```
   ❌ Реєстрація закрита на вихідних.
   ```

2. **П'ятниця після 16:00:**
   ```
   ❌ У п'ятницю прийом відповідей зупиняється о 16:00.
   ```

### Пріоритет перевірок

1. **Перша перевірка:** Адмін зупинив? → якщо так, повідомлення
2. **Друга перевірка:** Вихідні? → якщо так, повідомлення
3. **Третя перевірка:** П'ятниця після 16:00? → якщо так, повідомлення
4. **Інакше:** Бот працює ✅

## Технічні деталі

### Збереження стану

Стан бота зберігається у файлі `bot_state.json`:

```json
{
  "active": true
}
```

**Важливо:**
- Файл створюється автоматично при першому запуску
- Стан зберігається навіть після перезапуску бота
- Не видаляйте цей файл (він у `.gitignore`)

### Безпека

- Тільки користувачі з ID у `ADMIN_USER_IDS` можуть виконувати команди
- Інші користувачі отримають:
  ```
  ❌ У вас немає прав для виконання цієї команди.
  ```

## Поширені питання

### Що робити, якщо забув зупинити бота?

Бот автоматично зупиниться:
- У п'ятницю о 16:00
- На вихідних

Але для впевненості краще:
```
/admin_stop
```

### Як дізнатись, скільки заявок прийнято?

1. Відкрийте Google Sheets таблицю
2. Вкладка "Категорія-1" — лічильник в колонці A
3. Вкладка "Категорія-3" — лічильник в колонці A

Або додайте формулу в таблиці:
```
=COUNTA(A2:A)
```

### Що якщо хтось спробує надіслати заявку під час зупинки?

Він отримає:
```
❌ Прийом заявок закритий, до наступного тижня.
```

Заявка **не збережеться**.

### Чи можна змінити текст повідомлення?

Так, відредагуйте файл `bot_appscript.py`:

```python
if not bot_state.is_active():
    return False, "❌ Прийом заявок закритий, до наступного тижня."
    # Змініть текст тут ↑
```
